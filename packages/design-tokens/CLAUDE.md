# Package-Specific Constraints

- `tokens.json` is the single source of truth. Never hand-edit `dist/`, `lib/design_tokens.dart`, `dist/css/variables.css`, or `src/generated/tokens.ts` — they are regenerated by `pnpm build` (`codegen:css`, `codegen:flutter`, `codegen:ts`, then `tsc`).
- Generated outputs (`dist/`, `lib/design_tokens.dart`, `src/generated/tokens.ts`) are checked in. After editing `tokens.json`, run `pnpm build` and commit the regenerated files together.
- Consumers: `apps/desktop`, `apps/landing-page` (Tailwind preset + CSS vars), `apps/mobile` (Dart constants), `apps/mobile-v2` (`@zapengine/design-tokens/tokens` static TS module — no `node:fs` at runtime). Adding a new token requires updating `tokens.json` only — the codegens fan out from there.
