# Package-Specific Constraints

- `tokens.json` is the single source of truth. Never hand-edit `dist/`, `lib/design_tokens.dart`, or `dist/css/variables.css` — they are regenerated by `pnpm build` (`codegen:css`, `codegen:flutter`, then `tsc`).
- Generated outputs (`dist/`, `lib/design_tokens.dart`) are checked in. After editing `tokens.json`, run `pnpm build` and commit the regenerated files together.
- Consumers: `apps/frontend`, `apps/landing-page` (Tailwind preset + CSS vars), `apps/mobile` (Dart constants). Adding a new token requires updating `tokens.json` only — the codegens fan out from there.
